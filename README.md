# 📊 怪物彈珠抽卡分析器 (Monster Strike Gacha Analyzer)

這是一個為《怪物彈珠》玩家設計的進階抽卡預測工具。不同於傳統的「單抽實況模擬器」，本工具採用 **蒙地卡羅方法 (Monte Carlo method)**，在瀏覽器端瞬間執行 **1,000,000 次** (預設) 大數據平行模擬，試圖近似計算包含「非洲玉」保底機制在內的抽卡機率分佈。

幫助玩家在抽卡前進行較為科學的寶珠資源規劃，減少完全「瞎抽」的情況。

---

## ✨ 核心功能 (Features)

本工具包含兩大分析模式：

1. **模式一：目標達成所需抽數 (累積分佈函數 CDF)**  
   - **情境**：「我想抽到艦隊，大概需要準備多少寶珠？」  
   - **功能**：輸入目標隻數，系統會透過大量模擬計算出：
     - 25% 抽到目標的抽數（幸運落點）
     - 50% 抽到目標的抽數（中位數）
     - 85% 抽到目標的抽數（安全準備線）
     - 99% 抽到目標的抽數（極端非酋線）
   - **視覺化**：提供「達成率 vs 抽數」的曲線圖 (CDF)，清楚顯示抽數增加時，累積達成率如何提升，也能看出非洲玉觸發帶來的陡升區間。

2. **模式二：預期獲得數量 (機率密度函數 PDF)**  
   - **情境**：「我手上現在有 250 顆寶珠（50 抽），預期可以抽到幾隻目標角色？」  
   - **功能**：輸入預計抽數，系統會估算在該抽數下：
     - 抽到 0、1、2、3… 隻目標角色的機率
     - 整體的數學期望值（平均可得幾隻）

## 💡 常見機率迷思 (FAQ & Tips)

### 1. **雙 UP 卡池機率可以直接相加嗎？**
---
**Q: 卡池裡有兩隻目標角色 A、B，各自 1.2%。如果我想「抽到 2 隻」，是不是可以把機率 1.2% + 1.2% = 2.4%，然後目標數量設 2？**

**A: 要看你的「目標定義」是什麼：**

1. **情境一：你只想要「任意兩隻」這類角色**  
   例如：A / B 你都 OK，結果可能是：
   - 2 隻 A  
   - 2 隻 B  
   - 1 隻 A + 1 隻 B  
   在這種情況下，A 與 B 對你來說是「等價」的中獎事件，**可以近似視為總機率 2.4%，目標數量 = 2** 來做估算。

2. **情境二：你一定要「A 一隻 + B 一隻」都抽齊**  
   這種情況下：
   - 抽到第 1 隻 A 之後，A 對你來說就變成「溢出」，真正有用的只剩 B 那 1.2%。
   - 等於你前期機率是 2.4%，但中途會忽然掉回 1.2%。  
   這在統計上屬於「集齊問題 (Coupon Collector's Problem)」，**不能單純用 2.4% 和『兩隻』來替代**，實際需要的抽數會比「任意兩隻」多很多。

👉 **目前這個工具的「目標數量」設定，是針對「同一隻角色的累計 copies」而設計的。**  
如果要精準分析「湊齊多隻不同角色」，需要另外寫專門的多目標模擬邏輯。

## 🛠 技術細節 (Technical Details)

- **核心演算法**  
  - 使用 JavaScript 在瀏覽器端實作 Monte Carlo Simulation。  
  - 每次運算預設執行：  
    - 模式一：約 1,000,000(預設) 次模擬「抽到目標 N 隻所需抽數」  
    - 模式二：約 1,000,000(預設) 次模擬「在固定抽數下抽到的目標隻數」

- **機率模型（可依實際卡池調整）**  
  - 常規五星出率：12%  
  - 四星出率：88%（每次四星，星星碎片 +1）  
  - 星星碎片滿 50 觸發「非洲玉」，保證五星。  
  - 非洲玉中的目標角色命中率：  
    \[
    \text{目標非洲玉機率} = \frac{\text{目標常規機率}}{12\%}
    \]
    例如：目標常規 1.2%，則非洲玉中為 10%。
